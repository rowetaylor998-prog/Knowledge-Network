# Contributing

This project welcomes contributions to knowledge content, frontend code, backend code, documentation, governance notes, and AI experiment plans.

## Contribute Content

- Add or improve Markdown files under `content/` or `docs/`.
- Keep writing clear, structured, and useful for learners.
- Use headings, short descriptions, and TODO sections when creating placeholders.
- Cite sources when writing factual content, historical claims, technical explanations, or political-economic analysis.
- Mark uncertain claims clearly instead of presenting them as settled facts.

## Contribute Code

- Keep implementation simple and close to the MVP goal.
- Prefer small, focused changes over large rewrites.
- For frontend work, use the existing React/Vite structure in `apps/web/`.
- For backend work, use the existing FastAPI structure in `backend/`.
- Do not expose API keys or secrets in frontend code, commits, logs, or documentation.
- Run relevant checks before opening a pull request when possible.

## Submit Issues

Open an issue when you find:

- Incorrect or unclear knowledge content.
- Broken navigation, frontend bugs, or backend API problems.
- Missing citations or weak factual claims.
- Ideas for new learning paths, guided works, or AI provider support.

Include a clear title, a short explanation, and links or screenshots when useful.

## Basic Rules

- Keep knowledge open.
- Cite sources when writing factual content.
- Avoid low-quality spam, filler, and promotional content.
- Respect the platform's independence principles.
- Do not add vendor lock-in when a small abstraction can keep the project portable.

## Pull Request Workflow

1. Fork the repository or create a branch.
2. Edit content or code.
3. Commit with a clear message.
4. Open a pull request explaining what changed and why.
